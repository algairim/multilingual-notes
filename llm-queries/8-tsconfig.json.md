# LLM Query

Generate complete `backend/tsconfig.json` and `frontend/tsconfig.json` files.

# LLM Response

Here are the complete `tsconfig.json` files for both the backend and frontend, tailored to their specific environments.

### `backend/tsconfig.json`

This file is configured for a **NestJS** server environment. It's the standard file generated by the NestJS CLI, set up to compile CommonJS modules for a Node.js runtime.

```json
{
  "compilerOptions": {
    "module": "commonjs",
    "declaration": true,
    "removeComments": true,
    "emitDecoratorMetadata": true,
    "experimentalDecorators": true,
    "allowSyntheticDefaultImports": true,
    "target": "es2021",
    "sourceMap": true,
    "outDir": "./dist",
    "baseUrl": "./",
    "incremental": true,
    "skipLibCheck": true,
    "strictNullChecks": false,
    "noImplicitAny": false,
    "strictBindCallApply": false,
    "forceConsistentCasingInFileNames": false,
    "noFallthroughCasesInSwitch": false
  },
  "include": ["src/**/*"]
}
```

-----

### `frontend/tsconfig.json`

This file is configured for a **Vite + React** project. It's set up for a browser environment and assumes TypeScript is used for type checking only (`"noEmit": true`), while Vite handles the bundling.

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "useDefineForClassFields": true,
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "skipLibCheck": true,

    /* Bundler mode */
    "moduleResolution": "bundler",
    "allowImportingTsExtensions": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "react-jsx",

    /* Linting */
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true
  },
  "include": ["src"],
  "references": [{ "path": "./tsconfig.node.json" }]
}
```

-----

### `frontend/tsconfig.node.json`

**Important:** The `frontend/tsconfig.json` above requires this companion file in the **same `frontend/` directory**. This file is specifically for type-checking Node.js-based configuration files like `vite.config.ts`.

```json
{
  "compilerOptions": {
    "composite": true,
    "skipLibCheck": true,
    "module": "ESNext",
    "moduleResolution": "bundler",
    "allowSyntheticDefaultImports": true,
    "strict": true
  },
  "include": ["vite.config.ts", "tailwind.config.js"]
}
```
